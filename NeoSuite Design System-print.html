01

Filosofi Desain

NeoSuite dirancang untuk profesional yang bekerja berjam-jam dengan data padat — auditor, analis, akuntan. Setiap keputusan tunduk pada empat prinsip. Pakai ini sebagai filter saat membangun aplikasi baru di atas template.

P1

Kepadatan terkendali

Tampilkan banyak informasi tanpa berantakan. Baris 28px, padding rapat, ukuran font 13px dasar. Ruang putih dipakai untuk mengelompokkan, bukan memboroskan layar.

P2

Hierarki yang tenang

Chrome navy yang stabil, kanvas abu-abu netral, aksen biru hanya untuk aksi & status. Warna jenuh disimpan untuk makna (status), bukan dekorasi.

P3

Angka yang bisa dipercaya

Semua angka pakai font monospace dengan tabular-nums dan rata-kanan. Kolom tersejajar sempurna agar mata bisa memindai dan membandingkan.

P4

Konsistensi via token

Tidak ada nilai ajaib. Warna, jarak, radius, bayangan semua berasal dari variabel CSS. Ganti token → seluruh aplikasi ikut berubah secara seragam.

07

Tema & Kepadatan

Tema dan kepadatan dikontrol lewat class pada <body> dan tersimpan di localStorage. Karena semua warna adalah token, mengganti tema cukup mengganti definisi variabel.

ClassEfek
body.darkMode gelap — meredefinisi --navy, --bg, --surface, --ink, dan latar semantik. Chrome menjadi nyaris hitam-biru.
body.denseKepadatan tinggi — baris 24px, topbar 42px, padding tampilan 10px, gap grid 9px.
body.reduce-motionAksesibilitas — mematikan transisi & animasi untuk pengguna sensitif gerak.
Saat membuat aplikasi baru, jangan menulis warna mentah. Selalu rujuk token (mis. color: var(--ink)) supaya mode gelap & kepadatan otomatis berfungsi tanpa kerja tambahan.
08

Pedoman Penerapan

Cara memakai NeoSuite sebagai template untuk aplikasi baru — pertahankan kerangka & token, ganti modul & data.

Lakukan

  • Pertahankan kerangka TopBar + Sidebar + SubBar + View di setiap aplikasi.
  • Pakai token --var untuk semua warna, jarak, radius, bayangan.
  • Susun konten sebagai Portlet/Panel dalam grid 12px.
  • Pakai monospace + tabular-nums untuk semua angka & ID.
  • Pakai warna semantik hanya untuk status (badge, pill, indikator).
  • Jaga kepadatan: baris 28px, font dasar 13px, padding rapat.

Hindari

  • Menulis hex/px mentah di komponen — merusak tema & kepadatan.
  • Gradien dekoratif, sudut sangat membulat, atau bayangan tebal.
  • Warna jenuh sebagai latar besar atau hiasan.
  • Font selain Inter (UI) & JetBrains Mono (angka).
  • Spasi longgar bergaya marketing — ini alat kerja, bukan landing page.
  • Emoji atau ikon dekoratif yang tidak menyampaikan makna.
Langkah membuat modul baru
1Daftarkan modul ke navigasi (grup + label + ikon) di indeks modul.
2Buat file tampilan view_namamodul.jsx yang mengembalikan grid Portlet/Panel.
3Pakai primitif siap pakai: Portlet, Panel, Stat, Badge, Btn, Tabs, tabel .dtbl.
4Hubungkan ke konteks aktif (engagement/klien) lewat hook bersama agar SubBar & data sinkron.
5Verifikasi di mode terang, gelap, dan padat — tanpa menambah CSS warna apa pun.
