NeoSuite AMS · Migration Runbook

Migrasi Namespace window → Impor ESM

Menutup gap P1 dari evaluasi kesiapan Claude Code: mengganti ~150 titik Object.assign(window,…) dengan graf dependensi ESM statis di bawah Vite — secara mekanis, bertahap, dan tanpa pernah membuat app "layar putih". Paket sudah dibuat di folder migration/; app asli tidak tersentuh.

PendekatanDual-publish
TargetVite + ESM
Risiko app asliNol (terisolasi)
EksekusiNode · Claude Code
00

Strategi: dual-publish

Inti yang membuat migrasi 190-berkas ini aman: codemod menambah, tidak menimpa. Setiap berkas tetap menulis ke window seperti sekarang, dan sekaligus mengekspor lewat ESM. Konsumen kini mengimpor secara eksplisit, tetapi bila satu simbol terlewat, runtime masih menemukannya via window. Hasilnya: konversi yang selalu hijau, bukan big-bang yang rapuh.

Sebelum — global ajaib
// ui.jsx (kaki berkas) Object.assign(window, { Badge, Btn }); // view_psak46.jsx function PSAK46View(){ return <Badge>…</Badge> // Badge dari mana? }
Sesudah — impor eksplisit
import { Badge, Btn } from './ui.jsx'; // …kode asli tak berubah… Object.assign(window, { Badge, Btn }); // tetap (jaring) export { Badge, Btn }; // view_psak46.jsx import { Badge } from './ui.jsx';
Kenapa aman

Tidak ada kode lama yang ditimpa — codemod hanya menyisipkan blok import di atas dan export di bawah. Bus imperatif runtime (window.__amsOpenSA, __amsOpenCopilot, COMPLIANCE_CONFIG, window.claude) sengaja dibiarkan apa adanya. Rollback = hapus folder migration/.

01

Isi paket

Semuanya berada di migration/. Berkas bertanda biru dihasilkan oleh codemod saat dijalankan.

migration/ codemod.mjs ← konverter AST (@babel/parser), jalan di Node package.json ← deps + skrip: dev · build · test · codemod vite.config.mjs ← satu titik entri · plugin-react · sourcemap README.md ← runbook teks index.html ← GENERATED · 1 <script type="module"> (bukan ~190 tag) src/ ← GENERATED · salinan app/* + import/export ESM main.jsx ← entri: CSS → semua modul (boot order) → app.jsx *.js / *.jsx ← modul hasil konversi (asli tetap di /app) styles.css
app asli /app — utuh root buildless — tetap jalan butuh Node ≥ 18
02

Cara kerja codemod

Baca boot order dari HTML

Urutan <script src="app/…"> di NeoSuite AMS.html menjadi sumber kebenaran urutan muat & daftar berkas.

Bangun peta simbol global (AST)

Deteksi pemilik tiap simbol: Object.assign(window,{…}), window.NAMA = …, dan namespace data (AMS · PROC · BO · FAC · FIRMFIN · FSGEN; pemilik = berkas pertama dalam load order).

Sisipkan impor via analisis scope

Identifier bebas — termasuk komponen JSX <Foo/> dan <I.x/> — yang ada di peta tetapi tak dideklarasi lokal → import { … } from './pemilik'. React → 'react'; ReactDOM → 'react-dom/client'.

Dual-publish + buat entri

Tambah blok export di kaki berkas (window writes dipertahankan). Hasilkan src/main.jsx (impor urutan boot) dan index.html Vite.

Codemod juga melaporkan tabrakan nama (footgun const styles dll — dipakai pemilik pertama + fallback window, tetap perlu tinjauan) dan global tak dikenal (kemungkinan simbol terlewat / global runtime baru).

03

Jalankan

# Node ≥ 18 — Vite perlu Node, tak bisa diverifikasi di preview HTML
cd migration
npm install
npm run codemod:dry     # laporan: tabrakan nama + global tak dikenal (TIDAK menulis)
npm run codemod         # tulis src/*, main.jsx, index.html
npm run dev             # Vite — verifikasi app identik di http://localhost:5180
npm run build           # bundel produksi → migration/dist
Catatan lingkungan

Paket ini tidak dapat dijalankan/diverifikasi di dalam preview ini — Vite & codemod butuh runtime Node. Jalankan di mesin Anda atau sesi Claude Code. Pass --dry dulu untuk membaca laporan sebelum menulis apa pun.

04

Verifikasi & rollback

  • npm run dev membuka app; ⌘K, sidebar, dan drawer SA berfungsi.
  • Konsol bersih — tak ada "X is not defined" (kalau ada → impor terlewat, cek laporan codemod).
  • Angka konsisten dengan versi root: PSAK 46/71/73 & materialitas identik.
  • npm run build sukses; npm run preview jalan.
  • Bila ada yang aneh: hapus folder migration/ — app asli tidak pernah tersentuh.
05

Setelah dev hijau — langkah lanjut

Urutan ini melanjutkan roadmap dokumen evaluasi, dan sebagian besar cocok diserahkan ke Claude Code sebagai PR kecil yang terverifikasi.

Lucuti lapisan window berkas demi berkas

Hapus Object.assign(window,…) dan ganti sisa baca window.X dengan import. Setiap berkas = satu PR kecil, selalu bisa diverifikasi karena import sudah ada.

Jaring uji Vitest untuk "mesin angka"

AMS_CANON (materialitas, rekonsiliasi WTB↔LK), PSAK 46/71/73 — rasio nilai-per-uji tertinggi, fungsi murni tanpa DOM.

TypeScript bertahap

Mulai dari canon + kontrak antar-modul, lalu meluas. Graf ESM dari langkah ini adalah prasyaratnya.

Hasil

Begitu graf ESM terbentuk: tree-shaking, lint, dan penelusuran "siapa memakai apa" menjadi mungkin; kelas bug tabrakan nama top-level hilang; dan urutan boot tak lagi bergantung pada urutan tag <script> — persis tujuan gap P1.
