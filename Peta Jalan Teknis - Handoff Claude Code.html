A

Daftar tugas berurut

Urutan dipilih agar tiap langkah membuka langkah berikutnya & menjaga "diff angka nol" terhadap versi root. Fase rekayasa (1–6) sebagian besar mekanis; fase produk (7–13) berdiri di atas skema §B. Setiap tugas menyatakan dep (prasyarat) & out (keluaran terverifikasi).

1
RekayasaBaseline hijau & verifikasi kit migrasi

Jalankan build/ (precompile) & migration/ (Vite + codemod dry-run). Patok angka kunci (materialitas, PSAK 46/71/73) sebagai snapshot rujukan untuk seluruh langkah berikutnya.

dep: —out: prod.html & Vite boot identik root~½ hari
2
RekayasaPecah 6 berkas raksasa per domain

canon.js→canon/{materiality,psak46,psak71,psak73,reconcile}.js; data.js→data/{clients,engagements,wtb,team,engines}.js; + styles/related/groupaudit/copilot. Pemecahan ini juga memetakan langsung ke batas tabel di §B.

dep: 1out: tiap berkas <600 baris · regresi nol~1 hari
3
RekayasaVite jadi sumber build + ESLint gate

Satu entri main.jsx menggantikan 188 tag <script>. Pasang eslint (0 warning) sebagai gerbang mutu yang dapat dipanggil agen.

dep: 2out: npm run build & eslint hijau~1–2 hari
4
RekayasaJaring uji "mesin angka" (Vitest)

Uji fungsi murni canon: materiality(), reconcile, psak46/71/73. Inilah gerbang regresi yang melindungi langkah 5–7 & migrasi data nanti.

dep: 2out: cakupan canon ≥80% · test hijau~2 hari
5
RekayasaLucuti window → ESM (per workspace)

Codemod tanam import/export, lalu hapus Object.assign(window,…) berkas-demi-berkas. Bus runtime imperatif (__amsOpenSA, useAmsPersist) sengaja dipertahankan.

dep: 3, 4out: tree-shake tanpa "is not defined"~2–4 hari
6
RekayasaTypeScript bertahap dari kontrak canon

Ketik permukaan paling bernilai: tipe WTB, FIG, hasil tiap engine — tipe-tipe ini menjadi kontrak bersama dengan skema DB di langkah 7.

dep: 5out: tsc --noEmit hijau di src/canon~3–5 hari
7
ProdukSkema DB + migrasi + seed dari data.js

Implementasikan 24 tabel §B (Postgres/Prisma). Konstanta di data.js (CLIENTS/ENGAGEMENTS/WTB/TEAM…) berhenti jadi runtime → jadi seed migrasi & fixtures uji. Tipe langkah 6 dipakai ulang sebagai model ORM.

dep: 6out: seed memuat 8 klien · 8 perikatan · WTB utuh~4–6 hari
8
ProdukAPI + ganti localStorage sebagai SSOT

REST/tRPC di atas skema. Ganti useAmsPersist/loadLS dengan hook ber-API ke tabel module_state; localStorage turun pangkat jadi cache offline. (Seam §C·A.)

dep: 7out: 2 peramban berbagi state perikatan~1–2 minggu
9
ProdukAuth, sesi & RBAC ditegakkan

Login nyata (OIDC / email+TOTP); peran USER.role ditegakkan di API & UI; ganti mock di view_settings.jsx. (Seam §C·B.)

dep: 8out: uji otorisasi negatif lulus~1 minggu
10
ProdukAI Co-pilot via proxy server + RAG

Pindahkan window.claude.complete ke endpoint server (kunci di server, auth, rate-limit, log). RAG atas dokumen perikatan; demoReply() jadi fallback. (Seam §C·C.)

dep: 9out: jawaban model nyata · tanpa kunci di klien~3–5 hari
11
ProdukKonektor nyata (OAuth + sync)

Ganti status statis di data_import.js dengan OAuth + job sync + webhook; gunakan field_mapping yang sudah dimodelkan. Urut nilai: Coretax→firmtax, Bank Feed→cashbank. (Seam §C·D.)

dep: 8out: ≥1 konektor tarik data ujung-ke-ujung~2–3 minggu
12
ProdukJejak audit append-only + ekspor nyata

Ganti pseudo-hash klien (view_platform3.jsx) dengan rantai hash server-side di tabel audit_log; ekspor PDF/XLSX & e-Meterai/e-Sign nyata. (Seam §C·E.)

dep: 8, 9out: jejak tak-dapat-diubah dari klien~1 minggu
13
ProdukPengerasan, observability & deploy

Enkripsi at-rest/in-transit; retensi 10 thn (ISQM); CI/CD; target deploy (kontainer + Postgres). Backup/restore teruji; pen-test dasar.

dep: 8–12out: CI deploy otomatis · DR teruji~1–2 minggu

Aturan SSOT

Tabel hanya menyimpan input. Saldo teradjustasi, materialitas, pajak tangguhan (PSAK 46), ECL (71), & portofolio sewa (73) tetap dihitung AMS_CANON dari WTB saat baca — jangan dipersist sebagai kolom otoritatif. Materiality/compliance boleh di-cache (kolom DERIVED) untuk performa, tetapi WTB tetap satu-satunya sumber kebenaran. Ini menjaga invarian yang sama yang sudah berlaku di prototipe.

-- contoh DDL inti (Postgres) — turunan langsung dari data.js CREATE TABLE engagements ( id text PRIMARY KEY, -- "ENG-2025-014" client_id text REFERENCES clients(id), partner_id text REFERENCES users(id), manager_id text REFERENCES users(id), type text, standard text, fy text, status text, phase text, progress int, deadline date, budget_hrs numeric, actual_hrs numeric, materiality bigint -- overall; rincian di materiality_runs ); CREATE TABLE wtb_accounts ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, engagement_id text REFERENCES engagements(id), code text, "group" text, name text, lead text, ly bigint, unadj bigint, aje bigint, -- adj = unadj+aje DIHITUNG, tak disimpan UNIQUE (engagement_id, code) );
C

Titik-titik sambung file per celah

Untuk tiap celah prototipe→produk: berkas & simbol persis yang disentuh, dan apa yang berubah di tiap titik. Karena setiap engine sudah menerima state via argumen & jatuh ke localStorage hanya sebagai default, perubahan bersifat terlokalisasi — bukan penulisan ulang.

C·A · DATAlocalStorage → API / basis data→ Tugas 7–8

Ganti SSOT dari peramban ke server. Hook persist & engine adalah dua satu-satunya seam — sengaja terpusat.

app/contexts.jsx · useAmsPersist · loadLS · usePersisted
Seam utama. Ganti baca/tulis localStorage dgn fetch ke module_state. → optimistic-update + sinkron server; key ams.v1.<k> ↔ module_key.
app/data.js · ghgEngine · socEngine · aupNarrate · pfiEngine
Sudah menerima exec via argumen; hapus fallback loadLS() internal — state selalu disuntik pemanggil dari server.
app/canon.js · figuresFromWTB(wtb)
Tetap. Terima wtb reaktif dari useAudit().wtb yang kini ber-sumber API — tanpa perubahan logika.
server/ (baru)
Endpoint CRUD untuk 24 tabel; map kunci persist → rute. Seed dari data.js (Tugas 7).
C·B · AKSESAutentikasi, sesi & RBAC nyata→ Tugas 9

Model peran sudah ada (USER.role) & dirujuk UI — tinggal menegakkannya di auth.

app/contexts.jsx · AuthContext · useAuth
Seam utama. Ganti USER statis dgn sesi/token nyata; sediakan can(perm) untuk gating rute.
app/data.js · USER · FIRM
Berhenti jadi konstanta → baris di users/firms; dipinjam hanya sbg seed.
app/view_settings.jsx · flash('Kata sandi diperbarui')
Ganti tombol mock (ganti sandi · 2FA · timeout · IP allowlist) dgn panggilan API nyata.
app/data_platform.js · seed LOGIN/MFA
Peristiwa login berhenti jadi seed → ditulis nyata ke audit_log oleh server auth.
app/app.jsx · viewFor()
Bungkus rute sensitif dgn cek peran; uji otorisasi negatif sbg DoD.
C·C · AICo-pilot: klien → proxy server + RAG→ Tugas 10

Panggilan LLM dipindah ke server agar kunci tak di klien; demoReply() jadi fallback offline saja.

app/copilot.jsx · window.claude.complete · demoReply()
Seam utama. Arahkan ke POST /ai/complete; simpan ke ai_conversations/messages; pertahankan demoReply sbg degradasi.
app/llm_providers.js
Pindahkan pemilihan provider/kunci ke konfigurasi server; klien hanya tahu endpoint.
server/ai/ (baru)
Proxy ber-auth, rate-limit, redaksi PII, RAG atas dokumen perikatan; log tiap prompt/keluaran.
C·D · INTEGRASIKonektor statis → OAuth + sync nyata→ Tugas 11

Kontrak integrasi (scope, webhook, pemetaan field) sudah dimodelkan — implementasi per konektor.

app/data_import.js · status:'connected' · field map
Seam utama. status & last_sync berasal dari tabel connectors, bukan literal; field_mapping dipakai apa adanya.

Tampilkan status & log sync nyata; tombol "Hubungkan" memicu OAuth.
server/integrations/ (baru)
OAuth + job sync + webhook; post ke modul target sesuai map (Coretax→firmtax, Bank→cashbank, Privy→opinion, SharePoint→workpapers).

Catatan handoff

Bus runtime imperatif (__amsOpenSA, __amsOpenCopilot, useAmsPersist) sengaja dipertahankan melintasi migrasi ESM — ia bukan utang teknis, melainkan kontrak event antar-modul. Hanya Object.assign(window,…) untuk komponen/data yang dilucuti ke import/export. Mulai tiap celah dari seam utama (baris ber-label) — sisanya mengikuti.
