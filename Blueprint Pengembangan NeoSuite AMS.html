01

Ringkasan Eksekutif & Visi Produk

NeoSuite AMS adalah platform tunggal yang menyatukan seluruh siklus hidup praktik audit sebuah KAP — dari pengembangan bisnis dan penerimaan perikatan, perencanaan dan pelaksanaan audit, sampai finalisasi opini, pelaporan regulator, dan operasi internal firma — di atas satu basis data yang konsisten dan dapat ditelusuri.

150+
Modul fungsional tersebar di 23 grup & 2 ruang kerja
~220
File sumber di app/, dirakit via <script>
1
Sumber kebenaran angka — AMS_CANON di atas WTB
0
Build step — Babel mentranspilasi JSX di browser
Inti

Taruhan produk: alih-alih membangun puluhan alat terpisah, AMS menempatkan setiap standar (SA, PSAK/SAK, SPAP, regulasi OJK) sebagai modul yang menarik angkanya dari satu Working Trial Balance lewat mesin kanonik. Satu jurnal penyesuaian audit (AJE) mengalir konsisten ke pajak tangguhan, konsolidasi, sewa, sampai laporan keuangan — tanpa angka yang di-hardcode terpisah.

SIAPA

Pengguna

Partner, manajer, dan tim audit KAP; fungsi mutu (EQR/SOQM); back-office firma (HR, keuangan firma, pengadaan, legal). Antarmuka sepenuhnya Bahasa Indonesia dengan format mata uang id-ID.

APA

Cakupan

Perikatan audit end-to-end, jasa non-audit (reviu, asurans, due diligence), kepatuhan SA/PSAK/SAK Syariah, pelaporan OJK/PPPK, plus ERP ringan firma (GL, payroll, fasilitas).

MENGAPA

Diferensiasi

Keterlacakan (lineage) angka lintas-standar dan keterhubungan modul-ke-standar — bukan sekadar kumpulan formulir. Setiap figur dapat ditelusuri ke akun WTB sumbernya.

BAGAIMANA

Filosofi teknis

SPA no-build yang sengaja sederhana untuk dirakit cepat: registry global, transpilasi di browser, persistensi localStorage. Dirancang agar mudah ditambah modul tanpa pipeline.

02

Prinsip Arsitektur

Tujuh prinsip yang menjadi kontrak tak tertulis di seluruh basis kode. Pelanggaran terhadapnya tidak menghasilkan error yang jelas — melainkan layar putih senyap atau angka yang tak sinkron.

P1

Single Source of Truth (SSOT)

Angka berasal dari window.AMS.WTB melalui AMS_CANON.*. Tidak ada salinan privat atau angka hardcode untuk nilai yang sudah ada di canon. Status kertas kerja berasal dari deriveWpStatus.

P2

SPA no-build

Satu file entry NeoSuite AMS.html memuat seluruh app via tag <script> berurutan. Babel mentranspilasi JSX di browser. Tidak ada bundler, tidak ada langkah kompilasi.

P3

Scope global bersama

Semua .jsx ditranspilasi ke scope global yang sama. Komponen dipublikasikan ke window di akhir tiap file; nama top-level harus unik lintas-file.

P4

Registry-driven navigation

Navigasi digerakkan data: MODULES → MODULE_INDEX → WORKSPACES/GROUP_WS. Menambah modul = mendaftar entri, bukan menulis ulang router.

P5

Boot order = urutan script

Lima fase berurutan: Data → Fondasi → Lintas-sektor → Modul Halaman → app.jsx. app.jsx selalu terakhir; script baru disisipkan sebelumnya.

P6

Akses context defensif

Modul bisa dirender di drawer/luar provider. Hook context selalu dipanggil dengan penjaga: const nav = (typeof useNav==='function') ? useNav() : (()=>{});

P7

Graceful degradation

Router viewFor() jatuh ke <ComplianceView> (bila ada config) atau <StubView> berdesain untuk modul tanpa view khusus — sistem tak pernah crash karena modul belum lengkap.

03

Stack Teknologi & Urutan Boot

Entry tunggal NeoSuite AMS.html berisi <div id="root"> dan rangkaian <script> yang dimuat dalam lima fase. Urutan tag = urutan boot = urutan dependensi.

Stack

React 18.3.1 (UMD, di-pin + integrity) · Babel Standalone 7.x untuk transpilasi JSX di browser · JS biasa (IIFE) untuk lapisan data · CSS variabel di app/styles_*.css · localStorage untuk persistensi. Tanpa npm, tanpa bundler, tanpa server di sisi front-end.

Lapisan Data

data.js mengisi window.AMS (fmt, rp, FIRM, USER, CLIENTS, ENGAGEMENTS, WTB, AJE, RISKS, TEAM, WORKPAPERS…). File data_*.js melakukan Object.assign(window.AMS, …) atau membuat namespace tersendiri (PROC · BO · FAC · FIRMFIN · FSGEN). canon.js & forensic_canon.js membentuk AMS_CANON — mesin hitung.

04

Lapisan Data & SSOT — AMS_CANON

Jantung diferensiasi produk. Working Trial Balance adalah sumber kebenaran untuk saldo akun; AMS_CANON menarik figur akuntansi by-code dari WTB sehingga satu perubahan AJE mengalir konsisten ke seluruh modul. Tidak ada angka yang di-hardcode dua kali.

AliranWTB → CANON → Modulsingle source of truth

Setiap akun WTB dipetakan ke pos akuntansi kanonik lewat WTB_MAP — satu tempat untuk mengubah lineage. Fungsi figuresFromWTB(wtb) menghitung nilai tercatat neto (aset tetap, takberwujud, ROU/sewa, CKPN, DTA) dari saldo adj (setelah AJE) atau unadj (sebelum AJE), lalu modul-modul standar (PSAK 46, 24, 71, 73, 65 …) menarik darinya.

// canon.js — lineage by-code, bukan hardcode const WTB_MAP = { dbo: { code: '2-2300', label: 'Liabilitas Imbalan Kerja', sign: -1 }, // PSAK 24 ckpn: { code: '1-1210', label: 'CKPN', sign: -1 }, // PSAK 71 rou: { code: '1-2300', label: 'Aset Hak-Guna', sign: 1 }, // PSAK 73 dta: { code: '1-2500', label: 'Aset Pajak Tangguhan', sign: 1 }, // PSAK 46 … }; // satu AJE di WTB → figur konsisten di semua modul yang memakai AMS_CANON const fig = AMS_CANON.figuresFromWTB(useAudit().wtb);

Pantangan

Jangan menyimpan salinan privat atau hardcode angka yang sudah tersedia di AMS_CANON. Pelanggaran SSOT menghasilkan angka tak sinkron antar-modul — bug paling halus dan paling mahal di sistem ini. Semua nilai dalam Rp juta kecuali data sewa (rupiah penuh, dinormalkan saat dipakai). Tarif PPh Badan 22% (UU HPP 7/2021).

05

Navigasi & Taksonomi Modul

Navigasi dua tingkat berbasis data. Ruang kerja (workspace) memberi konteks tingkat atas; grup memetakan modul ke ruang kerja; sebagian grup standar sengaja disembunyikan dari sidebar namun tetap dapat diakses.

TIER 1

Workspace

Dua ruang kerja: Perikatan (kerja audit per-engagement) dan Firma (operasi & tata kelola firma). WORKSPACES mendaftar grup mana milik ruang kerja mana; GROUP_WS membalik peta itu.

TIER 2

Grup & Modul

MODULES = array grup berisi item {id,label,icon,deep,tag}. MODULE_INDEX meratakannya jadi id → {…,group}. deep:true = punya view nyata; selain itu render stub berdesain.

HIDDEN

Grup Tersembunyi

HIDDEN_GROUPS (SA 200/500/700, PSAK & SAK, Syariah) tak muncul di sidebar agar ramping — namun tetap diakses via ⌘K, chip "Standar Terkait", atau Matriks Kepatuhan.

LINK

Modul ↔ Standar

RELATED_SA[id] memetakan tiap prosedur perikatan ke Standar Audit terkait (chip + drawer). LINEAGE[id] (related_modules) memetakan dependensi hulu/hilir antar-modul.

// icons.jsx — registry yang menggerakkan seluruh navigasi const WORKSPACES = [ { id:'engagement', label:'Perikatan', groups:['Engagement Workspace','Core Planning','Core Execution',…] }, { id:'firm', label:'Firma', groups:['Practice Operations','People & Compliance',…] }, ]; const HIDDEN_GROUPS = ['SA · Tanggung Jawab (200)','Akuntansi (PSAK & SAK)', …]; // chip standar terkait pada tiap prosedur RELATED_SA['materiality'] = [{ code:'SA 320', title:'Materialitas …', phase:'Perencanaan' }];
Navigasi

Selalu memanggil nav(id, { from:'modulIni' }) agar breadcrumb "kembali" (SubBar) dan follow-workspace bekerja. Drawer SA dibuka via window.__amsOpenSA({ code, title, view? }). Modul tak terdaftar di router otomatis jatuh ke <ComplianceView> atau <StubView>.

07

Anatomi Modul & Kontrak Pengembangan

Prosedur baku menambah satu modul halaman: empat langkah wajib yang merekatkan modul ke router, registry, dan boot order — plus empat kaitan opsional yang mengintegrasikannya ke ekosistem standar.

1

Buat file view Wajib

Buat app/view_<nama>.jsx → alias hook React unik (lihat §08) → function MyView(){…} → akhiri dengan Object.assign(window, { MyView }); agar dapat dipanggil router dan file lain.

2

Daftarkan di registry Wajib

Di app/icons.jsx, tambah { id:'myid', label:'…', icon:'<key di I>', deep:true } ke grup yang tepat dalam MODULES. Grup sudah dipetakan ke workspace via GROUP_WS.

3

Sambungkan route Wajib

Di app/app.jsx, tambah case 'myid': return <MyView />; di dalam viewFor().

4

Sisipkan script Wajib

Tambah <script type="text/babel" src="app/view_<nama>.jsx"></script> di NeoSuite AMS.html, sebelum app.jsx.

+

Data & angka Opsional

IIFE data_<x>.js yang Object.assign(window.AMS, …) atau namespace baru. Figur angka ditarik dari AMS_CANON — jangan hardcode.

+

Kaitan ekosistem Opsional

Chip standar: RELATED_SA['myid']. Dock hulu/hilir: LINEAGE['myid']. Checklist kepatuhan: COMPLIANCE_CONFIG['myid'] (modul tanpa view khusus otomatis dirender <ComplianceView>).

// app/view_myid.jsx — kerangka modul baru const { useState: useStateMY, useMemo: useMemoMY } = React; // alias unik! function MyView() { const nav = (typeof useNav === 'function') ? useNav() : (() => {}); // defensif const fig = AMS_CANON.figuresFromWTB(useAudit().wtb); // SSOT, bukan hardcode return <Portlet title="Modul Saya">…</Portlet>; // primitif dari ui.jsx } Object.assign(window, { MyView }); // publish ke global
08

Aturan Emas Anti-Tabrakan

Karena semua .jsx berbagi satu scope global, nama top-level bertabrakan lintas-file. Pelanggaran enam aturan ini menghasilkan layar putih senyap — periksa konsol untuk "Identifier already declared".

#AturanKonsekuensi bila dilanggar
1Alias hook React per-file, unik. Jangan const {useState}=React; polos. Pakai sufiks modul: const {useState:useStateR}=React;Layar putih · "Identifier 'useState' already declared"
2Jangan const styles = {} global. Beri nama spesifik (const riskStyles=…) atau inline.Breakage senyap · style salah modul
3Ekspor di akhir file: Object.assign(window, { NamaKomponen, helperLain });ReferenceError saat dirender
4app.jsx selalu paling akhir; script baru disisipkan sebelumnya.Modul "tak terdefinisi" di viewFor
5Jangan redefinisi nama yang sudah di window (cek MODULE_INDEX, primitif ui.jsx, key I, namespace data).Override senyap komponen/data lain
6Akses context defensif: const nav = (typeof useNav==='function') ? useNav() : (()=>{});Crash saat dirender di drawer/luar provider
Konvensi sufiks

Sufiks alias mengikuti nama modul: Risk → R, Materiality2 → M2, PSAK46 → P46, FSGen → FS, Workspace → WS, ui.jsx → UI. File raksasa dipecah ke view_<x>_parts.jsx dan diimpor sebelum file utamanya.

10

State, Persistensi & AI

State perikatan/firma mengalir lewat context; state lokal modul dipersistensi ke localStorage dengan namespace ketat. Lapisan AI (Copilot + insights) berdiri di atas data kanonik yang sama.

CONTEXT

Auth · Firm · Audit

contexts.jsx menyediakan useAuth · useFirm · useAudit · useNav · useNavFrom dan <AppProviders>. useAudit().wtb reaktif — perubahan AJE mengalir ke canon yang menerimanya.

PERSIST

localStorage bernamespace

usePersisted/useAmsPersist(key,init) → key ams.v1.<key>. Prefs app → ams.<key>. Checklist → ams.comp.<stdId>.*. Selalu try/catch JSON.

WP STATUS

Status kanonik

Status kertas kerja dari deriveWpStatus(ref,audit,firm) — bukan flag privat. Buka WP kanonik via openCanonicalWp(nav,ref).

AI

Copilot & Insights

copilot.jsx + ai_insights.jsx + ai_extract.jsx di atas llm_providers.js. Dibuka via window.__amsOpenCopilot(); membaca data kanonik, bukan menyalinnya.

Kontrol global

Hook imperatif untuk kontrol lintas-modul: __amsOpenSA(data) (drawer standar) · __amsOpenCopilot() · __amsSetSidebar() · amsApplyPrefs(s) · compliancePct(stdId). Gunakan ini alih-alih menyentuh state internal modul lain.
